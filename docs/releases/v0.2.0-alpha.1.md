# overdare-studio-mcp v0.2.0-alpha.1

> **CRUD 완성 마일스톤** — Create/Read/Update/Delete 가 Stable Tool Contract 위에서
> 실제 OVERDARE Studio 까지 end-to-end 검증된 첫 기능 마일스톤. + Capability Layer.
>
> ⚠️ alpha. 비공식 로컬 프로토콜이라 Studio 업데이트로 깨질 수 있음(`npm run replay` 로 감지).

## Highlights
- **CRUD 전부 라이브 검증** — create/update/delete 가 실 Studio 에서 create→수정→삭제→원복까지 원자적으로 동작.
- **Stable Tool Contract** — 모든 도구가 동일한 `structuredContent: ToolResult` 반환. 실패도 throw 없이 구조화.
- **안전한 쓰기 파이프라인** — write lock + TOCTOU 가드 + atomic write + 자동 rollback.
- **Capability Layer** — Studio 기능 지원 여부를 probe·캐시, 미지원 기능은 계약 형태의 capability 에러.

## 안정성 / 보안
- write lock, 외부 동시수정 감지(`WriteError("conflict")`), tmp→fsync→rename 원자적 쓰기.
- 백업 `.overdare-backups/`(고유명·작업종료 정리), `WriteError.rollbackComplete` + Recorder 롤백 기록.
- `hub.token.read` JWT 는 trace 에 `<redacted>`.
- update 는 default 없는 부분 스키마(미지정 필드 보존 — 부분 업데이트 footgun 회피).

## Recorder / Replay
- `OVERDARE_MCP_TRACE=1` + `OVERDARE_MCP_TRACE_DIR=.trace` → `rpc.jsonl`/`session.json`/`metadata.json`/`ovdrjm.diff`.
- `npm run replay <session.json>` — 응답 shape 비교로 프로토콜 드리프트 감지.
