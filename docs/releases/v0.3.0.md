# overdare-studio-mcp v0.3.0

첫 공개 릴리스. Claude 로 OVERDARE Studio 를 조작하는 비공식 MCP 서버다.

---

## 소개

`overdare-studio-mcp` 는 Claude Desktop / Claude Code 에서 OVERDARE Studio 를 직접 조작할 수 있게
해주는 MCP(Model Context Protocol) 서버다. *"Workspace 에 네온 큐브 만들어줘"*, *"이 파트 색을
빨강으로 바꿔줘"* 처럼 자연어로 지시하면 실제 Studio 월드에 인스턴스·스크립트를 만들고 수정하고
삭제한다.

내부적으로 Studio 가 여는 로컬 JSON-RPC 포트와 `.ovdrjm` 월드 파일을 사용한다. 모든 쓰기는
백업·검증·자동 롤백을 거치는 파이프라인을 강제하고, 비가역 작업(publish)은 다중 게이트로 보호한다.
모든 RPC 는 기록되어 나중에 재생·비교할 수 있다.

> ⚠️ **비공식 프로젝트입니다.** OVERDARE 와 무관하며 공식 지원이 아닙니다. 로컬 단일 사용자용이며,
> Studio 업데이트로 로컬 프로토콜이 바뀌면 동작이 달라질 수 있습니다(`npm run replay` 로 감지).

## v0.3.0 추가 기능

- **CRUD 완성** — `studio_create_part`, `studio_update_part`, `studio_delete`, `studio_add_script`.
- **자산 임포트** — `studio_import_model`(에셋 스토어 모델), `studio_import_image`(로컬 이미지, 경로 보안 검증).
- **Action Sequencer** — `studio_apply_action_sequence`.
- **Publish (가드형)** — `studio_publish`, 기본 dry-run + 4중 게이트.
- **Capability Layer** — `studio_capabilities`. 지원하지 않는 기능은 호출 시 구조적 에러로 보고.
- **안전 인프라** — 쓰기 직렬화 락, TOCTOU 가드, 원자적 파일 쓰기, 자동 롤백.
- **Recorder / Replay** — RPC 기록 및 프로토콜 드리프트 감지.
- **Stable Tool Contract** — 모든 도구가 동일한 구조화 결과(`structuredContent`)를 반환.

## 주요 특징

### Stable Tool Contract
모든 도구가 동일한 결과 형태를 반환한다 — `success`, `operation`, `affected`, `warnings`,
`data`, `metadata`, `error`. 실패도 예외를 던지지 않고 `success:false` + 구조적 사유로 반환하므로,
클라이언트가 결과를 일관되게 다룰 수 있다. v1.x 동안 깨지 않는 것을 목표로 한다.

### CRUD
인스턴스 생성·조회·수정·삭제와 스크립트 추가를 자연어로. 수정은 부분 업데이트(지정한 필드만 변경,
나머지 보존), 삭제는 자식 포함이며 서비스 인스턴스는 보호된다.

### Capability Layer
시작 시(또는 첫 사용 시) Studio 가 어떤 기능을 지원하는지 확인해 캐시한다. 지원하지 않는 기능을
호출하면 Tool Contract 형태의 capability 에러를 돌려준다. Studio 버전 차이에 대응하기 위한 계층이다.

### Recorder / Replay
`OVERDARE_MCP_TRACE=1` 으로 모든 RPC 를 기록하고, `OVERDARE_MCP_TRACE_DIR` 로 재생용 세션을 남긴다.
`npm run replay <session.json>` 은 기록을 라이브 Studio 에 재생해 **응답 구조(shape)** 변화를 비교한다.
데이터 변경은 무시하고 프로토콜 변경만 감지하므로, Studio 업데이트 후 회귀 확인에 쓴다.

### Atomic Write
`.ovdrjm` 저장은 임시 파일 기록 → `fsync` → 원자적 `rename` 으로 처리한다. 크래시나 부분 기록이
발생해도 독자는 항상 "이전 전체" 또는 "이후 전체" 상태만 본다.

### TOCTOU Guard
쓰기 직전, 읽은 시점의 파일 지문(mtime+size+sha256)과 현재 지문을 비교한다. 그 사이 다른 프로세스가
같은 파일을 수정했으면 덮어쓰지 않고 `conflict` 로 중단해 외부 변경을 보존한다.

### Publish 4-Gate
`studio_publish` 는 비가역(외부 공개) 작업이라 **기본이 dry-run**이다. 실제 발사는 네 조건이 모두
충족돼야 한다 — capability 지원 + `OVERDARE_ALLOW_PUBLISH=1` + `confirm:true` + `dryRun:false`.
하나라도 빠지면 RPC 를 호출하지 않고 미충족 게이트를 보고한다. 실수 발사를 0 에 가깝게 만드는 설계다.

## 안전장치

- **쓰기 파이프라인** — 모든 `.ovdrjm` 변경은 `백업 → 수정 → 검증 → 적용 → 저장` 순서를 강제하며,
  어느 단계든 실패하면 자동 롤백한다.
- **쓰기 직렬화** — 동시 변이를 한 줄로 줄세워 파일/카운터 경쟁을 막는다.
- **백업 관리** — 작업별 백업을 전용 폴더에 두고 종료 시 정리(누적 방지). `--unsafe` 일 때만 생략.
- **경로 보안** — 파일 경로 입력(이미지/시퀀서 JSON)은 realpath 해소 + 프로젝트 경계 + 확장자 검증으로
  Path Traversal / Symlink 이탈을 차단한다.
- **민감정보** — 인증 토큰류는 trace 에 `<redacted>` 로 기록된다.

## 제한 사항

- **비공식 로컬 프로토콜** — Studio 업데이트로 깨질 수 있다. `npm run replay` 로 회귀를 확인할 것.
- **로컬 단일 사용자 가정** — Studio 가 실행 중이고 월드가 열려 있어야 동작한다(읽기/쓰기).
- **쓰기 도구는 프로젝트 폴더 필요** — `OVERDARE_PROJECT_CWD` 가 `.umap`/`.ovdrjm` 폴더를 가리켜야 한다.
- **publish 는 비가역** — 롤백할 수 없으며 replay 대상에서 영구 제외된다.
- **Studio 버전 보고 없음** — capability 의 `studioVersion` 은 현재 `null`(런타임 버전 조회 수단 부재).
- **스크린샷** — 현재 Viewport 모드만 지원.
